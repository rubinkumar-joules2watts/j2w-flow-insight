Scope of Work

Kick-off date13 April 2026 Engagement period13 Apr – 6 Jul 2026
Duration12 weeks · 3 months Production go-live6 July 2026 (Wk 12)

The current GEHC Axone LTS release process is manual and iterative — each release takes 2–3 months with up to 20+ promotion iterations per microservice across 27 services. Builds fail repeatedly because the LTS process treats each build as a fresh isolated build rather than a controlled promotion, causing cascading missing-dependency failures.

1  ·  SBOM-based dependency promotion

Run DSL Library (SYFT CLI) across all components · extract all dependencies deterministically including transitive · promote all to axone-electronic-record repositories in Artifactory before any LTS verification build is triggered.

2  ·  Per-component LTS verification

Trigger LTS verification builds per component only after Phase 1 promotion is complete · create LTS branches · monitor build outcomes · update registry on successful verification.

3  ·  Automated failure remediation

Deterministic log parsing for residual missing artifacts · automated targeted artifact promotion and build re-trigger · circuit breaker with configurable retry threshold (3–5 attempts) · structured failure report for any component that cannot be auto-resolved.

Overall project oversight by Vineet Shukla, CTO, ensuring delivery quality and strategic alignment. Formal Change Request process for any scope, design, or commercial deviations. No changes executed without written sign-off from both parties.

Safety Controls

Retry Limit
3–5 attempts per component. Pipeline halts promotion retries for that component. Component marked LTS failure; structured failure report emitted.
Circuit Breaker
Activated when retry limit exceeded. Failures surface via structured report; pipeline continues to next component.
No Manual Gate in Pipeline
By design — failures surface via structured report. Pipeline continues to next component automatically.
Promotion Scope Constraint
axone-electronic-record repos only. No promotion to any other repository without explicit config change.
